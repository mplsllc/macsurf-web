## Prompts for the essay

1. **When did it start?**
   - First commit context. Why a Mac OS 9 browser, not something more sensible.
2. **What triggered it?**
   - The specific frustration / observation that made this a project rather than an idle thought. (Hint: the modern web outgrew Classic Mac roughly twenty years ago and modern HTTPS killed it for good around 2016; a stock G3 today can barely reach a single live website.)
3. **What was the breakthrough moment?**
   - The first time MacSurf rendered real CSS on real hardware? The first JavaScript page? The fixes156 defensive-clamp bug that closed a five-year red herring? Pick one and describe what made it pivotal.
4. **Current state and why anyone should care.**
   - What MacSurf can do today (CSS3 + ES5 + PNG + native HTTPS via the sibling macSSL project), why this is a real browser rather than a curiosity, and what the next year looks like.

## Tone

Confessional / engineering-blog, not marketing. The README on github already sets the voice ("Built with stubbornness for a 25-year-old operating system.") — match that.
