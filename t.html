Intrinsic sizing — flex / grid / table content-sizing

Each OUTER box has a fixed width (heavy border). Compare each to Chrome. The question in every case: do items/columns size to their CONTENT, or collapse/clip/blow out?

1. flex — item with a long unbreakable token

1a. single flex item, content = one long word, container 300px
Chrome: the item can't shrink below its longest word (min-content), so it holds that width; the word stays readable. BROKEN if the item collapses to a sliver and clips, or drops to zero width.
